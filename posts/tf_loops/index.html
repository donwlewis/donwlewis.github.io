Looping in Terraform

Don Lewis
2020-04-24  about 551 words  3 min 

Well it has been a while since I have done anything in Terraform. My last venture into Terraform was over a year ago when they were on some 0.11.x version, and early beta of 0.12.0 was leaking out. At the time, it didn’t seem a good fit for what I was trying to do at the time, so I had made the decision to go with writing straight golang code. Since then, it seems some of the hang ups I had with Terraform have been solved. Especially looping.

Looping has definitely changed since version 0.11.x. Looping seems a lot better! After playing around with it, I was able to produce some sample code of how you would loop through a map object list:

provider "azurerm" {
  version         = "=2.0.0"
  subscription_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  features {}
}

variable "resource_groups" {
  type = map(object({
    location = string
  }))
  default = {
    "group01" = {
      location = "West US"
    },
    "group02" = {
      location = "East US"
    },
    "group03" = {
      location = "Central US"
    },
  }

}
resource "azurerm_resource_group" "terraform" {
  for_each = var.resource_groups
  name     = each.key
  location = each.value.location
}

From this basic example, you can see that you can easily looping through a map object using the for_each functionality within the resource. I am also able to access the key of the map object but by each.key. If I were to have a map object that was only a set of key/value pairs, I could easily access the value by using each.value:

provider "azurerm" {
  version         = "=2.0.0"
  subscription_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  features {}
}

}
resource "azurerm_resource_group" "terraform" {
  for_each = {
      group01 = "West US"
      group02 = "East US"
      group03 = "Central US"
    }
  name     = each.key
  location = each.value
}

This would probably not be the best way to utilize this, but it is another example of how loops work in Terraform 0.12.x.

Another thing to note is that this for_each expression does not seem to work on list objects. For that it seems like you are stuck with leveraging the old count expression. To achieve the same output from above, but with leveraging the count expression, our code would look something like this:

provider "azurerm" {
  version         = "=2.0.0"
  subscription_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  features {}
}

variable "resource_groups" {
  type = list(object({
    name     = string
    location = string
  }))
  default = [
    {
      name     = "group01"
      location = "West US"
    },
    {
      name     = "group02"
      location = "East US"
    },
    {
      name     = "group03"
      location = "Central US"
    },
  ]
}

resource "azurerm_resource_group" "terraform" {
  count    = "${length(var.resource_groups)}"
  name     = "${lookup(var.resource_groups[count.index], "name")}"
  location = "${lookup(var.resource_groups[count.index], "location")}"
}

This relies on the fact that your index matches exactly what you are looking for. This code is not as neat as the previous code, but it gets the job done.

There are other instances when you will probably need to leverage count vs for_each. HashiCorp provides a decent explanation here .

This type of functionality has got me excited about looking at Terraform again. Looping was such a pain in the older versions of Terraform, and this is much nicer and cleaner. I can also being to see how this can be converted into modules, and leveraging tfvars files to make this more modular. More updates to come as I discover more!
